What's in v2.0

Full repository migration. Merge requests to pull requests. Branch renaming. Batch operations. Everything you need for a complete GitLab → GitHub move.

Roadmap

1

Foundation

Infrastructure
  • ◆GitLab & GitHub API clients — authenticated REST clients with custom domain support for self-hosted & Enterprise instances
  • ◆Rate limiting & retry — exponential backoff respecting platform rate limits
  • ◆In-memory API caching — cached user profiles, PR lookups, and search results
  • ◆Project configuration model — source/target pairs, migration options, state tracking
2

Git Repository Migration

Core
  • ◆Auto-create GitHub repos — private by default via API
  • ◆Mirror push — full history, all branches and tags
  • ◆Branch renaming — master → main or any custom name
  • ◆Trim stale branches — clean up branches on GitHub no longer present in GitLab
3

Merge Request → Pull Request

Data
  • ◆MR → PR translation — open, closed, and merged merge requests become pull requests
  • ◆Comment migration — discussions and inline comments with author attribution
  • ◆Closed/merged reconstruction — temporary branches to faithfully recreate closed PRs
  • ◆Age filtering & skip rules — migrate only recent MRs, skip broken ones gracefully
  • ◆Idempotent re-runs — embedded IDs prevent duplicates on repeated runs
4

Batch & Scaling

Scale
  • ◆CSV project lists — define hundreds of source/target pairs in a single file
  • ◆Concurrent workers — parallel migrations with configurable concurrency
  • ◆Loop / daemon mode — continuous migration until cancelled
  • ◆Rich progress reporting — live progress bars and summary tables
5

CLI & Polish

UX
  • ◆gl2gh repo-migrate — single command: clone → push → rename → migrate MRs → convert CI/CD
  • ◆gl2gh batch-migrate — CSV-driven multi-project orchestration
  • ◆Resume support — interrupted batch runs pick up where they left off
  • ◆Web UI & docs updates — repo migration form, updated documentation
